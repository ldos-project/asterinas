# howdone

A command-line tool that runs a command and captures its output, along with provenance information, to a directory.

## Usage

```
usage: howdone [-h] [-c CONFIG] [-n NAME] [-d DIR | -p PREFIX] cmd ...
```

## Description

The provenance information is based on a configuration file that specifies a set of commands to run. The output of each
command is captured and stored in the output directory. All the commands are run with stdout and stderr combined to keep
them aligned. Each output file begins with the command that generated it. By default, commands are run after the main
command from the command line (so they can access the output). However, by providing `when` for the command, it
can run before the main command (see below for an example).

## Configuration

By default, `howdone` searches upward from the current directory for a file named `.howdone.yaml`, stopping at the
user's home directory. Only the first file found is used.

```yaml
# A prefix applied to the autogenerated name of output directories
prefix: run
# The file for the output of the main command
output_file: output.txt
# How the output directory is communicated to commands. Any or all sub-options may be set:
#   variable: set the named environment variable for all commands
#   argument: append argument(s) with `{OUTPUT_DIR}` substituted to the main command only (string or list)
#   change_directory: run all commands with cwd set to the output directory
output_dir:
  variable: OUTPUT_DIR
# A map of filenames to the commands which will be executed to fill them.
commands:
  dirty.diff: git diff --stat HEAD
  git-status.txt: git status
  uname-all.txt: uname --all
  start_date.txt:
    command: date
    when: before
```

## Output

The generated directory contains the files specified in the configuration and a file `meta.yaml` with the following
structure:

```yaml
run_name: <run name, if any, from command line>
current_directory: <absolute path of the current directory when running all the commands>
date: <date and time in ISO format, e.g. 2026-06-05T20:50:12>
full_command: <full command line of the howdone invocation>
meta_config:
    <the configuration used by howdone>
```

## Options

| Option | Description |
|--------|-------------|
| `cmd` | The command to run. If this is a single argument it is passed to a shell (as if via `sh -c 'cmd'`), otherwise arguments are passed directly to the subprocess, bypassing the shell. |
| `-h`, `--help` | Show help message and exit. |
| `-c CONFIG`, `--config CONFIG` | Provide a configuration file. If not provided, searches upward from the current directory for `.howdone.yaml`, stopping at the home directory. |
| `-n NAME`, `--name NAME` | Provide a name for the run. Stored in `meta.yaml`. |
| `-d DIR`, `--dir DIR` | Output to the given directory. |
| `-p PREFIX`, `--prefix PREFIX` | Prefix for the output directory name. Overrides the prefix in the config file. |
